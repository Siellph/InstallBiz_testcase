# Сервис скачивания и анализа каталога файлов

<details>

<summary>Задание</summary>
  
1. Страница скачивания
- На любой странице сервиса разместите кнопку «Скачать данные», которая запускает процесс скачивания документов через API.
- Процесс продолжается до тех пор, пока ручка имён не вернёт пустой ответ — вам нужно самостоятельно понять, когда каталог скачан полностью.
- Приветствуется визуализация показателей:
  - время старта процесса по Новосибирску (НСК);
  - «получено N названий файлов, скачиваю / скачано M из N».

2. Страница скачанных файлов и расчётов
- Списком отобразите скачанные файлы: имя файла и время его скачивания.
- Сортировка по времени скачивания.
- Пагинация.
- Выбор файлов: несколько точечно, все на странице, вообще все.
- Кнопка «Произвести расчёты» для выбранных файлов.

## Структура проекта

```
project/
  .env.example                  # шаблон конфигурации для развертывания
  run.py                        # точка входа
  requirements.txt
  app/
    __init__.py                 # фабрика приложения (create_app)
    config.py                   # настройки (base URL, candidate id, БД, таймзона)
    storage.py                  # слой хранения (SQLite): скачанные файлы
    api_client.py               # клиент внешнего API
    download_service.py         # оркестрация полного скачивания (фоновый поток)
    calculations.py             # подсчет статистики цифр
    routes/
      download.py               # роуты страницы и API скачивания
      files.py                  # роуты страницы и API файлов/расчетов
    templates/
      base.html                 # общий layout
      download.html             # страница "Скачивание"
      files.html                # страница "Скачанные файлы"
    static/
      css/style.css
      js/download.js            # запуск скачивания + опрос прогресса
      js/files.js               # список, пагинация, выбор, расчеты
```

#### Для управления настройками приложения используется файл `.env` в корне проекта. Создайте его на основе шаблона `.env.example`:

```bash
cp .env.example .env
```

#### Откройте созданный файл `.env` и заполните параметры:

```
EXTERNAL_API_BASE_URL=# укажите адрес API сервера 
CANDIDATE_ID=# Оставьте пустым, если хотите идентифицироваться по IP-адресу
DATABASE_PATH=# по умолчанию `app_data.sqlite3`
REQUEST_TIMEOUT=# по умолчанию`30`
DISPLAY_TIMEZONE=# по умолчанию Asia/Novosibirsk
```

## Запуск

```bash
# установите зависимости проекта
pip install -r requirements.txt
```

Убедитесь, что файл `.env` создан и настроен. Приложение автоматически подгрузит переменные окружения из него.

```bash
# запустите сервер
python run.py
```

Откройте `http://localhost:5000/` — страница скачивания,
`http://localhost:5000/files` — список скачанных файлов и расчеты.

## Как это работает

- **Скачивание** (`download_service.py`) крутится в фоновом потоке:
  циклично дергает `GET /api/files/names`, пока не придет пустой
  список, скачивает партиями по 3 файла через `POST /api/files/download`,
  сохраняет содержимое в SQLite и отмечает файлы через
  `POST /api/files/downloaded`. При 429 ждет `Retry-After` и повторяет
  запрос; при 403 (блокировка) ждет указанное время (или 30 минут по
  умолчанию) и продолжает — с точки зрения пользователя процесс просто
  продолжится позже, интерфейс покажет актуальный статус через опрос
  `/api/download/status`.

- **Файлы и расчеты** (`routes/files.py`, `calculations.py`):
  список скачанных файлов с пагинацией и сортировкой по времени
  скачивания, выбор отдельных файлов / всех на странице / вообще всех,
  расчет количества вхождений каждой цифры — суммарно по выбранным
  файлам и отдельно по каждому файлу.

> [!IMPORTANT]
> ## Ограничения текущей реализации
>
> - Прогресс скачивания хранится в памяти процесса.
> - SQLite использован ради простоты теста.
